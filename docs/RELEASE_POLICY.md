# Release Policy

سياسة الإصدارات المعتمدة لمنتجات **LAFRYHI GPT Store**.

## Versioning Rules

- `1.0.0` = أول إصدار
- `1.0.1` = إصلاحات
- `1.1.0` = ميزات جديدة
- `2.0.0` = تغييرات كبيرة

## Interpretation

### Patch

رقم مثل `1.0.1` يعني:

- إصلاحات صغيرة
- تحسينات في النصوص
- تعديلات لا تغيّر الوظيفة الأساسية

### Minor

رقم مثل `1.1.0` يعني:

- إضافة ميزات جديدة
- تحسينات واضحة في السلوك أو المحتوى
- بقاء التوافق العام مع النسخة السابقة

### Major

رقم مثل `2.0.0` يعني:

- تغيير كبير في الهيكل
- تغيير في طريقة الاستخدام
- احتمال كسر التوافق مع النسخ السابقة

## Practical Rules

1. يبدأ كل منتج من `1.0.0`.
2. أي تغيير صغير جدًا يأخذ `patch`.
3. أي ميزة إضافية بدون كسر الأساس تأخذ `minor`.
4. أي إعادة تصميم جذرية تأخذ `major`.
5. لا يتم تغيير رقم النسخة دون توثيق سبب التغيير.

## Store Usage

- `Version` في ملفات المنتج يجب أن يتبع هذه السياسة.
- عند إضافة منتج جديد، ابدأ بالنسخة المناسبة لمرحلة الاستقرار.
- عند نشر تحديثات للمتجر، اذكر النسخة بوضوح في الوثائق الداخلية.
