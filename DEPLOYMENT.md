# Deployment Guide

هذا الدليل يشرح كيفية رفع **LAFRYHI GPT Store** على دومينك الخاص باستخدام استضافة ثابتة.

المشروع الحالي:

- صفحات HTML ثابتة
- CSS و JavaScript بسيطان
- بيانات من `data/gpts.json`
- لا يحتاج build step

## قبل الرفع

1. تأكد أن الموقع يعمل محليًا عبر خادم، مثل:

```powershell
python -m http.server 4173
```

2. راجع الروابط التالية:

- `index.html`
- `gpts.html`
- `gpt.html?slug=...`
- `about.html`
- `contact.html`

3. تأكد أن `data/gpts.json` محفوظ بصيغة صحيحة وأن الصور أو الأيقونات المؤقتة تعمل.

## الرفع على Netlify

### الطريقة السريعة

1. افتح Netlify وأنشئ Site جديدًا.
2. اسحب مجلد المشروع أو ادفعه إلى GitHub ثم اربطه.
3. اجعل مجلد النشر هو جذر المشروع نفسه.
4. اترك `Build command` فارغًا.
5. انشر الموقع.

### بعد النشر

1. افتح `Site settings`.
2. أضف `Custom domain`.
3. اتبع خطوات DNS التي يعرضها Netlify.
4. انتظر اكتمال تفعيل SSL.

## الرفع على Vercel

### الطريقة المعتادة

1. أنشئ مشروعًا جديدًا داخل Vercel.
2. اربط مستودع GitHub الخاص بالموقع.
3. اختر إعداد Static أو `Other` إذا ظهرت لك الخيارات.
4. لا تضف أي build command.
5. اجعل المخرجات من جذر المشروع.
6. انشر الموقع.

### بعد النشر

1. افتح إعدادات المشروع.
2. أضف الدومين الخاص بك.
3. حدّث سجلات DNS لدى مزود النطاق وفق التعليمات.
4. تحقق من أن الصفحة الرئيسية تُفتح مباشرة من الدومين.

## الرفع على GitHub Pages

### طريقة النشر

1. ارفع المشروع إلى مستودع GitHub.
2. افتح `Settings > Pages`.
3. اختر النشر من الفرع الرئيسي ومن الجذر.
4. احفظ التغييرات.
5. انتظر حتى يظهر رابط Pages.

### عند استخدام دومين مخصص

1. أضف ملف `CNAME` في جذر المشروع إذا كان ذلك مناسبًا لطريقتك في النشر.
2. أو اربط الدومين من إعدادات GitHub Pages مباشرة.
3. اضبط سجلات DNS لدى مزود النطاق.

## إعدادات DNS للدومين

غالبًا ستحتاج إلى واحد من النمطين التاليين:

- `A record` أو `ALIAS` للدومين الرئيسي
- `CNAME` للنسخة `www`

اتبع القيم التي يطلبها مزود الاستضافة الذي اخترته، لأن القيم تختلف من منصة لأخرى.

## التحقق بعد النشر

افحص هذه النقاط بعد الربط:

1. الصفحة الرئيسية تعمل بدون أخطاء.
2. صفحة GPTs تعرض البطاقات بشكل صحيح.
3. صفحة التفاصيل تفتح من `slug`.
4. ملف `gpts.json` ما زال يُقرأ من الاستضافة.
5. الموقع يستجيب جيدًا على الهاتف والحاسوب.

## توصية عملية

إذا أردت أقل تعقيد ممكن، استخدم Netlify أولًا.  
إذا أردت ربطًا مباشرًا مع GitHub ونشرًا تلقائيًا، استخدم Vercel.  
إذا أردت أبسط خيار مرتبط بالمستودع نفسه، استخدم GitHub Pages.
